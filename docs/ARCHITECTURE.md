# 架构设计

## 总体架构

```text
智能眼镜摄像头
  -> 本地抽帧、缓存匹配、快速显示
  -> Go 服务端
  -> 云端视觉 API 或自训练模型
  -> 返回标签、单词、解释、例句
  -> 眼镜显示和发音
```

## 模块职责

### DDD 服务端分层

```text
interfaces/httpapi
  接收眼镜请求，转换 JSON DTO，返回 HUD 可用的中英标签。

application
  编排 RecognizeFrame 用例：校验帧、查缓存、调用识别、生成学习内容、保存场景。

domain
  表达核心业务：Frame、SceneRecognition、VisualObject、LearningCard、BoundingBox。

infrastructure
  提供可替换实现：内存场景仓储、Mock 视觉识别器、静态中英词典。
```

### 眼镜本地端

- 采集视频帧。
- 控制上传频率。
- 使用本地缓存复用历史识别结果。
- 在画面上显示 `A/B/C` 标签。
- 播放英文发音和短句。
- 离线时使用最近结果和本地轻量模型。

### Go 服务端

- 接收图片帧或视频抽帧。
- 做 scene hash 去重。
- 管理本地缓存、区域缓存、历史识别结果。
- 调用云端视觉 API 或本地模型。
- 返回结构化英语学习数据，包括 `display_text` 和 `speak_text`，方便眼镜直接展示和朗读。

### AI 识别层

第一阶段建议使用云 API 或现成模型快速验证体验。

第二阶段再收集眼镜真实场景数据，训练专用检测模型，并导出 ONNX、NCNN 或 TensorRT 版本用于本地端。

## 缓存策略

### 场景缓存

对整帧生成 hash 或 embedding。相似度高时，直接复用上一次识别结果。

当前第一版已经实现基于 `scene_hash` 的内存缓存。客户端传入 `last_scene_hash` 时，服务端会优先返回缓存结果，并标记 `from_cache: true`。

### 区域缓存

比较当前帧和上一帧，只对变化区域重新识别。适合手进入画面、物品移动、视角小幅变化。

### 预加载

摄像头缓慢移动时，根据运动方向预估下一帧可能出现的区域，提前请求云端识别。

### 离线模式

网络不可用时：

- 优先复用相似场景缓存。
- 使用本地轻量模型识别常见物体。
- 低置信度结果标记为 `maybe`。

## 延迟目标

- 本地缓存命中：50ms 以内。
- 服务端 Mock/本地模型：100ms 到 300ms。
- 云端识别：500ms 到 2000ms。

眼镜端应该先显示缓存结果，再异步用云端结果修正。
